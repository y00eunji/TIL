# 회고

`drag` 와 관련된 이벤트들을 다뤘다. drag 이벤트 중 drop을 사용하려면 dragover 이벤트와 같이 사용하지 않으면 이벤트가 먹지 않는 다는 것을 새로 배웠다. dragend 이벤트를 사용해서 드래그이벤트 종료 시 이벤트를 발생 시켜 한 이벤트 안에서 모든 걸 해결 할 수 있었지만 드래그를 진행하고 있는 요소와 드래그를 끝냈을 때 드롭할 수 있는 요소를 둘 다 저장해야 한다는 번거로움이 있어 drop을 사용하는 쪽으로 바꿨다. 아직 정확히 왜 두 이벤트가 동시에 필요한 지에 대한 공부는 필요한 것 같다. 아직 추측으로는 drop 이벤트가 발생하기 위해서는 drag가 시작했다는 것을 이벤트로 감지를 하고 있고 그 후에 drop 이벤트를 처리해줄 수 있기 때문인것 같다. 

또한, dragover 사용 시 특정 DOM이나 전체 window에 드래그, 드랍 이벤트를 막고 싶은 경우 e.preventDefault를 사용해주면 된다. 

오늘은 새로운 이벤트들과 동적으로 html을 추가하는 작업을 반복적으로 했지만 아직 많이 익숙하지 않은 것을 보니 많은 연습과 생각이 필요한 것 같다. 

이번 프로그래밍은 리팩토링을 고려해 진행을 하다 보니 무지성을 코드를 작성하지 않고 전체적인 순서를 짜고 진행을 하다보니 논리에 맞지 않은 코드들을 계속해서 수정하는 작업을 동시에 하면서 기록을 하고 있어 다음 리팩토링에 도움이 될 것 같다. 


어쩌다 과거에 내가 같은 코드를 작성한 걸 발견했는데 그때와 지금의 큰 차이가 없지만 앞으로 리팩토링을 진행할 때 발전을 느낄 수 있을 것 같다. 

오늘 크게 느낀 점은 함수 로직을 먼저 분리하고 미래의 리팩토링에 너무 신경을 쓰다보니 시간이 많이 소요되고 밑단이 되는 코드가 없다보니 머리속으로 생각해야해서 구조를 짜는 데 어려움을 겪었다. 앞으로는 먼저 밑단이 되는 코드를 작성하고 리팩토링을 진행해야지 시간이 단축될 것 같다. 또한, 로직이 너무 길어질때는 반대로 생각해보는 방법도 좋은 것 같다.
